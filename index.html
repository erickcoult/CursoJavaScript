Variáveis

Uma variável é um espaço na memória do computador destinado a um dadoo seu nome que é alterado durante a execução do algoritmo. Para funcionar corretamente, as variáveis precisam ser definidas por nomes e tipos

Entretanto, há uma regra para se dar um identificar para a variável, sendo elas:

  • Podem começar com letras, $ ou _
  • Não podem começar com números
  • É possível usar letras ou números
  • É possível usar acentos e símbolos
  • Não podem conter espaços
  • Não podem ser palavras reservadas, ou seja, palavras que seriam códigos para o JavaScript
  • Inteiros: números inteiros (0, 1, 2 , 3…)
  • Reais: números inteiros e decimais (0; 1,2; 2,1; 3)
  • Caracteres: números reais, letras e outros símbolos
  • Lógico: comandos de VERDADEIRO ou FALSO

Um exemplo para a criação de um var seria: var exemplo = 14. Assim, quando colocarmos "exemplo" no terminal, ele nos dará "14" como resposta

Tratamento de dados

Tratamento de dados nada mais é do que fazer com que o JS processe as informações dadas nas prompt's (janelas de interação). Ou seja, fazer uma soma, divisão, conversão e etc.

Para isso, precisamos criar uma variável, dar a identificação e então, a pergunta. Devemos ter a quantidade de janelas para a quantidade de números que vamos somar. Ou seja, se eu irei somar 2 números, vou ter que criar 2 janelas de prompt.

Além de criar uma Variável para fazer a soma, preciamos criar uma Terceira variável que teria como a finalidade de fazer a soma das outras duas. Também precisamos criar um alerta para nos mostrar o valor.

Assim, o resultado ficaria:

  • var n1 = Number.parseFloat(window.prompt ('número 1'))
  • var n2 = Number.parseFloat(window.prompt ('número 2'))
  • var s = n1 + n2
  • window.alert('a soma dos valores é ' + s)

Podemos perceber que no código há uma "parseFloat". Ela tem como função de fazer que o JS apenas somará números quebrados. Há também outros dois, sendo eles " parseInt" e o próprio "Number". Assim, temos:

  • parseFloat: trabalhará apenas com valores quebrados, que possuam vírgula.
  • parseInt: trabalhará apenas com valores inteiros.
  • Number: O próprio JS decidirá qual irá utilizar.

var p1 = olá! var p2 = tudo bem ? var final = p1 + p2 final = olá!tudo bem?

    Atribuição simples

    Outra coisa que é semelhante a matemática é a Atribuição simples. Ela nada mais é do que atribuir um valor à uma variável. Ou seja, uma função. Veja os exemplos:

    Auto-atribuição

    a Auto-atribuição tem como objetivo de dar uma "nova" atribuição à variável. Logo, se você deu um valor à variável e, logo em seguida, dá uma nova atruibuição, aquela ultima substituirá a primeira. Veja abaixo:

    var n = 3 => o "n" vale 3. n = n + 4 => o "n", que antes valia 3, agora passa a valer 7. n = n - 5 => o "n", que antes valia 7, agora passa a valer 2. Também podemos simplificar var n = 3 n += 4 n -= 5

    Relacionais

    Relacionais são operadores da grande maioria das linguagens das programções, sendo mostrado por símbolos matemáticos

    • > |maior que
    • < | menor que
    • >= |maior ou igual que
    • <= |menor ou igual que
    • == |igual à
    • != |não igual/diferente
    • === |igualdade restrita, ou seja,tendo o mesmo valor e do mesmo tipo (typeof)

    Ao utilizarmos um Operador Relacional, nosso resultado será sempre um valor booleano, ou seja, verdadeiro(true) ou falso(false)

    5 > 2 => true 8 <= 5 =>false 4 != 4 => false 7 == 7 => true 2 === '2' => false "2" === "2" => true

    Operadores lógicos

    No JavaScript possuimos 3 Operadores lógicos, sendo eles

    • ! => Negação
    • && => Conjunção
    • || => Disjunção

    O operador de Negação é tratado como Operador UNÁRIO, ou seja, após inserimos !, resultado será ou false ou true

    Já o operador de Conjunção é considerado como BINÁRIO, assim, possuindo doisoperadores lógicos( um de cada lado). Um exemplo é quando você pede uma caneta vermelha euma azul, ou seja, você só ficará satisfeito se tiver uma caneta azul e uma vermelha

    Os operadores de Disjunção também são considerados BINÁRIO que possuem dois operadores lógicos. Seguindo o exemplo de canetas, as Disjunção busca ou vermelha ou azul. Assim, tendo uma das duas, você ficará satisfeito. Logo, basta apenas um true para o resultado ser verdadeiro

    Uma coisa que devemos saber é que os operadores lógicos também possuem uma order de grandeza, sendo primeiro o NÃO (Negação), depois o E (Conjunção) e po ultimo o OU(Disjunção)

    IMPORTANTE: Quando se tem Operadores Relacionais, Operadores Lógicos e Operadores Aritméticos na mesma expressão, é seguido primeiro por ARITMÉTICO, RELACIONAIS E LÓGICOS.

    var a = 5 var b = 8 a > b && b % 2 == 0 a > b && 8 % 2 == 0 O lado direito é verdadeiro já que o resto de 8 % 2 é igual a 0 5 > 8 && true false && true Logo, o resultado final será falso já que os dois tinham que ser true

    Operador Ternário

    Constituído apenas por dois símbolos, sendo o ? e :. Ele possui este nome por conter 3 partes sendo

    teste ? true : false

    Com isso, podemos fazer uma expresão que se der um determinado valor (verdadeiro ou falso) no bloco de teste, aparecerá o valor inserido no bloco true:false. Veja abaixo

    var x = 4 var res = x / 2 == 2 ? 5:8 Nesta situação, caso o resultado de x/2 for 2, então o valor de res será 5(true), senão, mostrará 8(false) var res = 4/2 == 2 ? Assim, o valor de res é 5, já que a expressão do bloco teste é verdadeira res = 5

    Outro exemplo muito usado seria fazer o Operador Ternário como um "seletor de dados"

    var idade = 19 var res = idade >= 18 ? 'MAIOR':'MENOR' res = 'MAIOR'

    Condição Multipla

    A condição multipla tem sua função de fornecer um valor fixo. Ou seja, ela nao tem apenas a possibilidade de sim ou não, tendo vários outros caminhos.

    Ela é representada pelo comando switch(expressão){}. Podemos notar que é dada uma expressão e não uma condição como vemos em if.

    Dentro do bloco, podemos adicionar os "case" e o "default. Cada case possui o seu bloco que irá conter as expressões desejadas. Já o default funciona como um else, sendo algo opcional. Logo, caso os valores de CASE sejam insatisfeitos, ele fará o default

    switch (expressão){ case valor1: break case valor2: break case valor3: break default: break

    ATENÇÃO:após cada CASE, usamos o código break. O comando break encerra o loop atual, switch , ou o loop que foi informado e transfere o controle da execução do programa para o comando seguinte. Ou seja, suponhamos que, ao testar a expressão (do switch) e o "CASE valor2" esteja correto, o break fará com que seja encerrado o fluxo, passando para o próximo código. Assim, caso não haja um break, o programa irá executar todos os comandos de todos os CASES até achar um break, bugando tudo.

    Repetição

    A estrutura de repetição, como pelo próprio nome já diz, tem a fução de repetir o comando até que se torne falso. Ou seja, ao inserimos uma condição dentro de um bloco, ela será verificada e, se der como uma condição VERDADEIRA, o programa voltará todo o processo novamente, fazendo então, um looping, até que a condição seja FALSA.

    Uma das estrutura de repetição que temos é a while (condição){}. Podemos ler o while como um 'ENQUANTO'.

    Para uma melhor clareza, imagine uma pizza com 24 pedaços. Podemos criar uma function com a expressão 'comerPizza' e, dentro dela, inserimos "comer um pedaço". Teriamos que inserir 24x o 'comer um pedaço'. Porém, com o While, podemos fazer isso

    (enquanto)While (temFatia()){ comerFatia

    Assim, enquanto tem fatia, o 'comerFatia' será executado. Depois, o programa rodará novamente 24x até não ter mais 'temFatia'.
